    关于ret2libc

    最近学这块感觉有点迷,打算记录点东西然后好好去看看书,补下基础233

    0x00 利用原理

    ret2libc 这种攻击方式主要是针对 动态链接(Dynamic linking) 编译的程序，因为正常情况下是无法在程序中找到像 system() 、execve() 这种系统级函数.而libc.so 是 linux 下 C 语言库中的运行库glibc 的动态链接版，并且 libc.so 中包含了大量的可以利用的函数，包括 system() 、execve() 等系统级函数

    0x01 攻击思路

    找到system函数的地址

    找到’/bin/sh’, ‘sh’等字符串的地址

    0x02 题目解析

    题目来源:angelboy

    1.Source code

    2.Makefile

    gcc -fno-stack-protector -mpreferred-stack-boundary=2 -m32 ret2lib.c -o ret2lib

    0x03 利用思路

    1.看下保护

    发现只开了NX保护,于是不能写入shellcode

    2.泄露思路

    观察代码发现,strcpy函数可以有栈溢出

    然后继续寻找发现,可以利用泄露puts函数地址,从而找到system函数的绝对地址

    利用公式

    A真实地址-A的偏移地址 = B真实地址-B的偏移地址 = 基地址

    于是

    system_addr - system_offset = puts_addr - puts_offset=libc的偏移量

    利用cyclic找到溢出的偏移是60,gdb的find指令寻找到sh的地址

    然后利用pwntools写exp
