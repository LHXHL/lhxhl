0x00 ROP

  • Gadget
    • read/write to reg/memory
      • pop edx ; pop eax ; ret
      • mov dword ptr [edx], eax ; ret
    • system call
      • int 0x80
    • change esp
      • pop esp ; ret
      • leave ; ret

0x01 题目分析

1.source code

2.Makefile

gcc -fno-stack-protector -m32 -static -mpreferred-stack-boundary=2 simplerop.c -o simplerop

0x02 利用思路

1.看下保护

buf只给了20的大小,而我们可以写入的有100,且只有NX保护,没有canary,存在栈溢出.

于是,我们就更据题目提示,我们利用

  • execve(“/bin/sh”,NULL,NULL)
    • write to memory
    • 将’/bin/sh写入’存储器中
    • 可以用两次,分别写入’/bin’和’sh\x00’

首先,readelf找到可写的.data段,(一般是这里可执行),.bss段有时也可,

要注意栈溢出是32个字节,我们剩下gadget的只有68,所以要尽量找短的

然后ROPgadget就用exp说明

0x03 EXP

前两段ROP就是将’bin/sh’字符串写入到data可执行的地方
